HTML Content and the Box Model

Within HTML files, various elements are given content to display, such as text, images, and video. Initally, this content is always displayed within an invisble and flexible box, with a width, height and depth defined by the size of the provided content. This initial placement of HTML content within boxes is what is known as the box model.

By default, the text you are reading now would begin from the very top-left corner of the page and stretch right across the full page-width before shifting to a new line. The box wraps immediately around the text without interfering with its shape or size.

The dimensional aspects of an HTML element (width, height and depth) are the foundation of the box model.

CSS Stylesheets and the Box Model

To bring greater structure, clarity and flow to webpages, web developers use CSS stylesheets to define how their HTML content is displayed, changing various aspects of their content such as the size, shape, colour, and alignment with the other surrounding elements.

The boxes that hold each standard paragraph throughout this blog post have been centered, and their width has been restricted to a maximum of 612 pixels; furthermore, the text itself has been justified and stylised, all with the code, below:
position: relative;
max-width: 612px;
text-align: justify;
line-height: 1.5em;
justify-content: center;

For the purposes of this blog post, the dimensional aspect you ought to focus on is max-width: 612px as this is the aspect that constricted the width of the box. Similar code can be deployed to control the height, substituting width for height:

Padding

Padding is the space between the content and its border, which is initally not included; set to zero. A web developer may elect to include padding between the content and its boundary/border/box with the padding keyword:

padding:
padding-top:
padding-bottom:

If defining multiple padding sizes for each edge of the box, web developers may also minimise the amount of code they must write by using 'padding shorthand', which defines all of the various padding properties on a single line of code:

padding: Top&Bottom, Left&Right;
padding: Top, Left&Right, Bottom;
padding: Top, Right, Bottom, Left;

Margins

Margins fulfil the same role to padding, from the other side of the border. Web developers use margins to create space between the borders of various elements by increasing the size of the box's outer edge beyond its border:

Summary

Content, padding, borders, and margin are the foundational aspects of the box model. Combined, they are utilised to efficiently and precisely arrange the various HTML elements present on any given webpage.
